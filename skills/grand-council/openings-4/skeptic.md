# Skeptic Opening Statement

**Counter to C1 (License-Mixing Integrity):** A build-time CI attribution check is superficially comforting but fundamentally insufficient for a single public GitHub repo combining these three sources. First, transitive redistribution of n2yo-derived TLEs—even if cached server-side as proposed in Session 3—risks violating Space-Track's upstream covenants against bulk redistribution if the repo's caching scripts or API contracts make it trivial for downstream consumers to bulk-scrape the cache. Second, the CC BY 4.0 attribution chain for GOGPT requires more than just a string check in a file; if GOGPT numbers are mathematically transformed (e.g., the Mathematician's capacity-weighted Lorenz curve), the license requires indicating that modifications were made, which a naive grep-based CI check cannot enforce. Third, if any SatDump configuration file, hardware mapping, or pipeline graph definition is copied directly into the repo rather than referenced as an external dependency, the GPL copyleft boundary is blurred, potentially tainting the entire repository.

**Counter to C2 (Reference Shelf vs. Service):** The claim that publishing a "reference shelf" (catalogs and schemas) rather than a service cleanly shifts liability to the reader is false. It does not eliminate liability; it merely obfuscates it. If the Emperor publishes a schema mapping that incorrectly drops GOGPT's status weighting, resulting in a downstream financial or political miscalculation, the failure originated in the shelf's transformation logic, not the downstream consumer's execution. Publishing the *script* that performs the ingest makes the Emperor liable for the *logic* of the ingest.

**Counter to C3 (Seams in the Shelf):** Bundling Expo/React Native mobile-integration seams (like TLE cache API contracts and hardware compatibility JSON) directly into the reference shelf violates separation of concerns. The reference shelf should represent the raw epistemic mapping of the domain (WGS84+UTC events, TLE formats, plant capacity schemas). A mobile API contract introduces application-specific implementation details (e.g., Expo's specific bridging requirements or React Native's async storage limitations) into a repository that is supposed to be a pure reference. **Alternative:** The shelf should export pure JSON schemas and GraphQL/OpenAPI definitions; a separate, downstream `grid-and-chain-sdk` repository should generate the React Native/Expo bindings from those schemas.

**Counter to C4 (Dashboard Display):** Surfacing GOGPT numbers on a live, fast-glance dashboard explicitly risks the status-weighting violation guarded against in Session 3. The failure mode: a dashboard's primary constraint is screen real estate. To make numbers legible at a glance, UI designers inevitably collapse complex distributions into single "headline" figures. Displaying "1,047 GW In-Development" prominently, even with a tiny, structurally required pie chart of the breakdown nearby, creates a visual hierarchy that overrides the epistemic nuance. The metric becomes the headline, violating the spirit, if not the letter, of the Session 3 ruling.

**Stance:** The reference shelf model is dangerously close to an SDK and risks both legal and epistemic contamination. 

**Motion to Modify (C3):** Strip all application-specific API contracts and mobile seams from the `conceptual-engineering-references` repository. Mandate that the repository publish only pure, implementation-agnostic data schemas (OpenAPI/JSON Schema), forcing any Expo/RN integration to occur in a separate downstream repository.
