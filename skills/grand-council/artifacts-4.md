# Session 4 — Distribution Posture

## Emperor's directive

Make the session-3 reference shelf (n2yo + SatDump + GOGPT) **consensable
and usable** — publishable to the masses as a public GitHub repo, then
surfaced as a pplx.app dashboard. Simultaneously scaffold the integration
seams the emperor's grid-and-chain-mobile app (Expo/RN) will plug into.

## Claims Under Review (C1..C4)

- **C1** — A reference shelf mixing proprietary+rate-limited (n2yo),
  copyleft (SatDump), and permissive+attribution (GOGPT) can be
  distributed as a single public GitHub repo without compromising any of
  the three license bindings, provided attribution + upstream ToS
  respect is enforced at build time.

- **C2** — The correct posture for "distributing to the masses" is a
  reference shelf, not a service — i.e. the emperor publishes catalogs,
  schemas, and scripts, and each downstream consumer runs their own
  ingest against upstream. This shifts liability from operator to reader.

- **C3** — Mobile-integration seams (TLE cache API contract, hardware
  compatibility JSON, WGS84+UTC event type) belong in the reference shelf
  itself, not in a separate SDK repo — the shelf IS the SDK for the
  emperor's portfolio apps.

- **C4** — A pplx.app dashboard surfacing the shelf must carry, on every
  view: (a) source with aggregator marked, (b) TLE epoch age for orbital
  data, (c) status breakdown for GOGPT capacity, (d) the CC BY 4.0
  attribution string. Anything less repeats the mistakes the session-3
  ruling explicitly guarded against.
